## Summary
AI pair programming extends traditional human-human pair programming by introducing an AI assistant as a collaborative coding partner. Unlike one-off code generation, it creates a sustained, interactive relationship between developers and AI that provides real-time suggestions, handles routine tasks, and amplifies developer productivity. Tools like GitHub Copilot (Codex-based) exemplify modern AI pair programmers.

## Key Concepts
- **AI Pair Programmer**: An AI assistant that continuously collaborates alongside a developer
- **Real-Time Suggestions**: Code completions, function generation, refactoring hints during active coding
- **Task Offloading**: Routine boilerplate, repetitive patterns, and scaffolding delegated to AI
- **Interactive Loop**: Developer writes intent → AI suggests implementation → Developer guides/refines
- **Skill Asymmetry**: Humans handle high-level design, AI handles low-level synthesis
- **Productivity Amplification**: 2025 data shows 30–50% faster development cycles with AI pair programming
- **Learning Opportunity**: Developers learn patterns and idioms from AI-generated code
