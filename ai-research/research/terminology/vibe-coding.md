## Summary
"Vibe coding" is a development style where engineers fully embrace AI assistance, writing code by giving high-level intent to LLMs (e.g., Cursor Composer with Claude 3 Sonnet) rather than manually implementing details. The developer sees something, says something, runs it, and iterates—"I don't really code, I just see stuff, say stuff, run stuff, and copy paste stuff, and it mostly works." Coined by Andrej Karpathy in February 2025, the term went viral and was named Collins English Dictionary Word of the Year 2025.

## Key Concepts
- **Intent-Driven Development**: Describe what you want ("decrease the sidebar padding by half") rather than finding and editing code
- **AI-as-Synthesizer**: LLM generates implementation details from high-level specs
- **Diff Acceptance**: Trust the AI enough to accept diffs without reading them line-by-line
- **Error-Driven Iteration**: Copy-paste errors into the AI ("error X happened") and let it fix them
- **Minimal Typing**: Voice input (SuperWhisper) + minimal keyboard navigation
- **Pragmatism Over Purity**: Code quality matters less than shipping speed; iterate in production
- **Vibes**: Operating on intuition and momentum rather than detailed planning

## Relevance to Generative AI Engineer Role
Vibe coding represents a philosophical shift: code as ephemeral artifact vs. code as persistent specification. As a Generative AI Engineer, you'll need to understand both the productivity gains and the risks. Vibe coding works for rapid prototyping but breaks down in systems requiring specification rigor. Your job includes designing architectures where vibe-coded components can safely coexist with formally-specified ones, managing the tension between speed and durability.
