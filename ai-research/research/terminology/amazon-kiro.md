## Summary
Amazon Kiro is an agentic IDE launched by AWS (July 2025) designed specifically for Spec-Driven Development. Rather than jumping directly to code generation, Kiro enforces three levels: requirements (specs), design (architecture), and implementation (code). Each level generates executable, version-controlled markdown artifacts. Kiro represents the industry shift from "vibe coding" to structured, formal specification-driven development at scale.

## Key Concepts
- **Spec-First**: Specifications become executable, first-class artifacts
- **Three-Level Thinking**: Requirements → Design → Implementation
- **Markdown Artifacts**: Each level creates versionable documentation
- **Automated Hooks**: Specifications trigger automated task and code generation
- **Traceability**: Changes in specs ripple through design and implementation
- **Reduced Drift**: Formal structure minimizes divergence between intent and code
- **Agentic IDE**: Integrated environment where agents work within SDD constraints
- **Executable Specifications**: Specs drive plans, tasks, and code scaffolding
