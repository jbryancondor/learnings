## Summary
The middle-loop, articulated by Martin Fowler in March 2026, is a feedback mechanism operating between immediate task execution (inner loop) and long-term strategic learning (outer loop). It captures situational feedback from deployed agents—patterns of mistakes, recurring decision points, edge cases—and uses it to improve agent behavior within a development cycle. The middle-loop bridges reactive correction and proactive architecture changes, enabling agents to improve through accumulated operational experience.

## Key Concepts
- **Three-Loop Model**: Inner (task execution), middle (operational feedback), outer (strategic learning)
- **Situational Feedback**: Data from agent operations during normal deployment
- **Pattern Recognition**: Identifying recurring issues from operational data
- **Fast Iteration**: Middle-loop cycles measured in days/weeks, not months
- **Tactical Improvement**: Fixes that don't require architecture changes
- **Behavior Refinement**: Adjusting agent decision-making based on real outcomes
- **Experience Accumulation**: Agents improve through deployed operations
- **Learning Without Retraining**: Modify behavior through prompts, tools, or guardrails
