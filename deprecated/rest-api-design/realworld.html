Real-World API Patterns

How leading companies have solved REST design challenges in production APIs used by millions of developers.

Stripe
The Gold Standard of REST
  • Expandable objects: ?expand[]=customer — related data in one request. Elegant N+1 solution.
  • Date-based versioning: pinned per account, override via Stripe-Version header. 100+ breaking changes, all still supported.
  • Idempotency keys: Idempotency-Key header caches result and returns on retry.
  • Internal governance: 20-page API design doc; cross-functional review for every endpoint.

GitHub
Pragmatic Embedding
  • Eager embedding: always-needed relations embedded inline (e.g. owner inside repository).
  • Link header pagination: RFC 8288 rel="next", rel="last".
  • Rate limit headers: X-RateLimit-Limit, -Remaining, -Reset. Returns 429 on exceed.
  • 5,000 req/hour for authenticated; 60 for unauthenticated.

Twilio
Consistency as a Feature
  • Consistent structure across all products — code feels familiar regardless of which API.
  • Documentation-driven design: write docs for sample apps first, then build.
  • Transversal API team for governance across product teams.
  • Treat docs like code: tested, reviewed before release.

Twitter / X
Fields & Expansions Pattern
  • ?fields= and ?expansions= for controlling response shape — similar to JSON:API sparse fieldsets.
  • Cursor-based pagination with next_token.
  • Tiered rate limits by access level (Free → Enterprise).

WordPress
HAL-Style Linking & Embedding
  • _links present by default — hypermedia references to related resources.
  • ?_embed opt-in: inline related data into a single response.
  • Clean separation: link by default, embed on demand.

Shopify
Cost-Based Limits + Date Versioning
  • Date-based API versioning (e.g. 2024-04) similar to Stripe.
  • REST Admin API deprecated October 2024; all new apps must use GraphQL.
  • Anti-pattern identified: including IDs of other objects instead of the objects themselves.
